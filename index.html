Capítulo 1

Noções gerais de programação em Python

Edital: Processamento de texto · Extração de características · Aprendizagem de máquina

Por que Python? Linguagem de propósito geral com vasta coleção de módulos para PLN (NLTK, spaCy). Sintaxe legível e curva de aprendizado suave.

  • Algoritmos e programas: algoritmo = sequência finita de instruções; programa = algoritmo codificado em Python.
  • Modos do interpretador: interativo (prompt >>>) e script (arquivo .py executado com F5 no IDLE).
  • Variáveis e tipos: int, float, str, bool. Separador decimal é ponto (.), não vírgula.
  • Sequências: strings (imutáveis), listas (mutáveis, [ ]), tuplas (imutáveis, ( )), conjuntos (set, sem ordem), dicionários (chave–valor, { }).
  • Controle de fluxo: testes lógicos (==, !=, <, >, in), condicionais (if/elif/else), loops for e while.
  • Funções: nomeadas (def) e anônimas (lambda). Parâmetros, return, variáveis locais vs. globais, recursão.
  • Módulos externos: import / from...import; collections.defaultdict, re, statistics, NLTK, spaCy.
  • Expressões regulares: metacaracteres (., *, +, ?, [], |, \b, \d, \w); re.findall, re.sub; raw strings r'...'.
  • NLTK: corpus, tokenizadores, etiquetadores; nltk.download() para dados.
  • spaCy: nlp = spacy.load('pt'); tok.pos_ (classe gramatical), tok.orth_ (forma), tok.lemma_ (lema).
  • Filtragem: compreensão de listas [x for x in lista if critério]; stop words = palavras funcionais removidas antes da análise.
  • defaultdict(int): dicionário com valor padrão 0; evita KeyError em contagens.
Capítulo 2

Análise quantitativa de corpus

Edital: Processamento de texto · KDT · Análise sintática

Corpus: conjunto de textos digitais coletados segundo critérios linguísticos, finito, representativo e opcionalmente anotado.

  • Tipos: oral vs. escrito; geral vs. temático; não-anotado vs. anotado; monolingue vs. paralelo.
  • Corpora no NLTK: Machado (obras literárias), Mac-Morpho (notícias + POS), Floresta Sintática, SUBTLEX-pt-BR.
  • Leitura de arquivos: open('arq.txt', 'r', encoding='utf-8'); .read(); .close().
  • Limpeza: strip(), split(), isalpha(), lower(); função limpar() personalizada.
  • Contagens: types = palavras distintas; tokens = ocorrências; riqueza lexical = types / tokens.
  • Hápax legômena: palavras com frequência = 1; indicam raridade e afetam modelos.
  • Estatística descritiva: média, mediana, moda, variância, desvio-padrão; módulo statistics.
  • TF-IDF: TF = freq. no doc / total de termos; IDF = log(N / df); TF-IDF = TF × IDF.
  • POS tagging: spaCy atribui NOUN, VERB, ADJ, ADV, DET, ADP a cada token.
  • Concordanciador: exibe contexto (KWIC) ao redor de expressão-alvo para análise qualitativa.
  • Gráficos: matplotlib.pyplot — plt.bar(), plt.pie(); parâmetros labels, autopct, title.
  • Stemmer: NLTK RSLPStemmer reduz palavras à raiz; pode gerar formas incorretas (hápax falso).
  • sorted() + lambda: ordenação de dicionários por valor; reverse=True para decrescente.
  • Variância amostral (n-1): stat.variance; variância populacional (n): stat.pvariance.
Capítulo 3

Modelos de n-gramas

Edital: Aprendizagem de máquina · Reconhecimento de voz · Gramáticas · Interpretação semântica

N-gramas: sequências de n unidades linguísticas extraídas de um texto: unigrama (n=1), bigrama (n=2), trigrama (n=3).

  • Probabilidade: P(E) ∈ [0, 1]; soma de todos os eventos = 1.
  • Probabilidade condicional: P(E|F) — chance de E dado F; base dos modelos de n-gramas.
  • Assunção de Markov: P(wₙ | w₁…wₙ₋₁) ≈ P(wₙ | wₙ₋₁); só o contexto imediato importa.
  • MLE: P(w₂|w₁) = C(w₁,w₂) / C(w₁); estimativa por contagem de n-gramas no corpus.
  • Suavização de Laplace (add-1): P_Laplace = (C + 1) / (C_contexto + |V|); evita P = 0.
  • Perplexidade: PP(T) = (1/P(p₁…pₙ))^(1/n); quanto menor, melhor o modelo.
  • Aplicações: previsão de palavras, correção ortográfica, tradução automática, reconhecimento de voz.
  • Corpus treino vs. teste: divisão típica 80/20; avaliação SEMPRE no corpus de teste.
  • Dados esparsos: maioria dos n-gramas possíveis nunca ocorre no corpus → muitos zeros.
  • Marcadores <s> e </s>: indicam início e fim de sentença para capturar padrões de fronteira.
  • <DES>: marcador para hápax legômena filtrados do vocabulário de treinamento.
  • tuple() em Python: ngramas(n, sent) retorna lista de tuplas usando range(len(sent) - n + 1).
  • Chomsky (1955): "Colorless green ideas sleep furiously" — gramatical mas semanticamente absurda; n-gramas podem atribuir P > 0 a ela.
Capítulo 4

Classificadores bayesianos ingênuos

Edital: Aprendizagem de máquina · Análise de sentimentos · KDT

Classificação: tarefa de atribuir um dado (documento, mensagem) a uma classe predefinida com base em atributos (features).

  • Regra de Bayes: P(A|B) = P(B|A) × P(A) / P(B); calcula probabilidade posterior a partir de dados observados.
  • Classificador ingênuo: assume independência condicional entre atributos dado a classe; ĉ = argmax P(c) ∏ P(fᵢ|c).
  • MLE para classificadores: P(fᵢ|c) = contagem(fᵢ, c) / contagem(c); P(c) = Nc / N.
  • Suavização (k): P = (C + k) / (C_total + k × |V|); evita P = 0 para palavras ausentes.
  • Logaritmos: somar log P em vez de multiplicar P evita underflow numérico.
  • Features: presença/frequência de palavras; vocabulário como espaço de atributos.
  • Treinamento supervisionado: modelo aprende P(c) e P(fᵢ|c) a partir de corpus anotado.
  • Avaliação: matriz de confusão (VP, FP, FN, VN); acurácia, precisão, cobertura (recall), medida-F.
  • Medida-F: F = 2 × (P × R) / (P + R); combina precisão e cobertura em uma métrica.
  • Acurácia: (VP + VN) / total; pode ser enganosa em classes desbalanceadas.
  • Precisão: VP / (VP + FP). Cobertura (Recall): VP / (VP + FN).
  • WSD (Word Sense Disambiguation): determinar o sentido correto de palavra ambígua pelo contexto (ex.: "manga" = fruta ou roupa).
  • Aplicações: spam, análise de sentimentos, POS tagging, classificação de textos.
Capítulo 1 – Noções gerais de programação em Python
Questão 01
Em Python, qual é a diferença fundamental entre uma lista e uma tupla?
Resposta: (b). Listas ([ ]) são mutáveis — permitem append(), alteração por índice e remoção. Tuplas (( )) são imutáveis; uma vez criadas, seus valores não podem ser modificados. Ambas suportam tipos mistos e indexação negativa.
Questão 02
Qual o resultado de re.findall(r'\b\w+\b', 'Olá, mundo!') em Python?
Resposta: (b). \b marca fronteira de palavra e \w+ captura sequências alfanuméricas. A vírgula e a exclamação são separadores, então retorna ['Olá', 'mundo'].
Questão 03
defaultdict(int) do módulo collections é útil para contagem de palavras porque:
Resposta: (b). defaultdict(int) inicializa qualquer chave nova com 0, permitindo contagem[palavra] += 1 sem verificar existência prévia. Muito usado para frequência de palavras em PLN.
Questão 04
Qual alternativa descreve corretamente uma função lambda em Python?
Resposta: (b). lambda par1, par2: expressão — define uma função anônima de uma linha, muito usada como argumento de sorted(), map() e filter().
Questão 05
Considere: palavras = ['casa', 'rato', 'gato']; print(palavras[-1]). O que é exibido?
Resposta: (c). Índices negativos contam a partir do final. palavras[-1] é o último elemento, 'gato'.
Questão 06
No NLTK, corpus.stopwords.words('portuguese') retorna:
Resposta: (b). Stop words são palavras de alta frequência e baixa carga semântica (de, a, o, que...). O NLTK fornece listas prontas por idioma para filtragem antes de análises de conteúdo.
Questão 07
Qual metacaractere de expressão regular casa exatamente 4 dígitos consecutivos?
Resposta: (b). \d corresponde a qualquer dígito [0-9] e {4} especifica exatamente 4 repetições. \w inclui letras e underscore; [0-9]+ casa um ou mais dígitos (não exatamente 4).
Questão 08
Em spaCy, o atributo tok.pos_ de um token retorna:
Resposta: (b). tok.pos_ retorna a etiqueta morfossintática (POS tag) universal. tok.orth_ retorna a forma ortográfica; tok.lemma_ retorna o lema.
Questão 09
Qual a diferença entre os modos interativo e script no interpretador Python?
Resposta: (b). No modo interativo (>>>), cada linha é interpretada e o resultado aparece imediatamente. No modo script (.py + F5), o arquivo completo é executado e resultados só aparecem com print().
Questão 10
O loop for x in range(2, 10, 3) itera sobre quais valores?
Resposta: (b). range(início, fim, passo) → range(2, 10, 3) gera 2, 5, 8. Começa em 2, avança de 3 em 3, para antes de 10.
Capítulo 2 – Análise quantitativa de corpus
Questão 01
Riqueza lexical de um corpus é calculada como:
Resposta: (b). Riqueza lexical = types / tokens. Quanto maior, mais diverso é o vocabulário em relação ao tamanho do texto.
Questão 02
O que é um hápax legômeno?
Resposta: (b). Hápax legômena são palavras com frequência absoluta 1 no corpus. Indicam raridade lexical e afetam modelos de n-gramas (dados esparsos).
Questão 03
O componente IDF do TF-IDF serve para:
Resposta: (b). IDF = log(N / df_t). Palavras como "de" e "a" têm IDF ≈ 0; palavras raras têm IDF alto, indicando maior relevância informacional.
Questão 04
Por que especificar encoding='utf-8' ao abrir arquivos de corpus em Python?
Resposta: (b). UTF-8 é o padrão Unicode capaz de representar todos os caracteres do português (ã, ç, é, etc.). Sem especificá-lo o Python usa a codificação do SO, podendo causar erros.
Questão 05
A medida estatística central resistente a valores extremos (outliers) é:
Resposta: (c). A mediana é o valor central da série ordenada e não é distorcida por outliers. A média é sensível a valores extremos.
Questão 06
O corpus Mac-Morpho disponível no NLTK é caracterizado por:
Resposta: (b). Mac-Morpho é um corpus de notícias da Folha de S. Paulo (1994) com anotação morfossintática automática revisada por linguistas.
Questão 07
Qual função separa as palavras de uma string pelo espaço, retornando uma lista?
Resposta: (b). str.split() divide pelo espaço (padrão) e retorna lista. strip() remove espaços nas bordas; join() concatena; replace() substitui substrings.
Questão 08
Em etiquetagem POS com spaCy, qual etiqueta corresponde a advérbios?
Resposta: (c). Tagset universal do spaCy: NOUN (substantivo), VERB (verbo), ADJ (adjetivo), ADV (advérbio), DET (artigo/determinante), PROPN (nome próprio), ADP (preposição).
Questão 09
A principal desvantagem do TF-IDF em coleções muito pequenas é:
Resposta: (b). Com poucos documentos, uma palavra relevante pode aparecer em todos eles, resultando em IDF = log(N/N) = 0, tornando o TF-IDF nulo mesmo para termos importantes.
Questão 10
O concordanciador (KWIC) no contexto de análise de corpus serve para:
Resposta: (b). O concordanciador apresenta a palavra buscada com N palavras de contexto à esquerda e à direita, permitindo análise do uso real da língua.
Capítulo 3 – Modelos de n-gramas
Questão 01
A Assunção de Markov, aplicada a modelos de bigramas, estabelece que:
Resposta: (b). No bigrama, P(wₙ | w₁…wₙ₋₁) ≈ P(wₙ | wₙ₋₁). A Assunção de Markov simplifica ao considerar apenas o contexto imediato (n-1 palavras anteriores).
Questão 02
A estimativa MLE para o bigrama P(menino | esse) é calculada como:
Resposta: (b). P_MLE(menino | esse) = C(esse menino) / C(esse). Divide-se a contagem do bigrama pela contagem do contexto (unigrama anterior).
Questão 03
Por que a suavização de Laplace é necessária em modelos de n-gramas?
Resposta: (b). Se um bigrama não ocorreu no treinamento, MLE atribui P = 0, zerando toda a probabilidade de qualquer sentença que o contenha. A suavização redistribui massa probabilística para cobrir n-gramas ausentes.
Questão 04
Perplexidade em modelos de linguagem é uma medida onde:
Resposta: (b). PP(T) = (1/P(p₁…pₙ))^(1/n). Avalia-se sempre no corpus de TESTE. Modelo bom → probabilidades altas no teste → perplexidade baixa.
Questão 05
Para que servem os marcadores <s> e </s> ao construir modelos de n-gramas?
Resposta: (b). <s> e </s> são convenções computacionais para fronteiras de sentença. No bigrama permitem modelar P(primeira palavra | início) e P(fim | última palavra).
Questão 06
Qual é o principal problema dos dados esparsos em modelos de trigramas?
Resposta: (b). O vocabulário V cria V³ combinações possíveis de trigramas. Mesmo corpora grandes cobrem uma fração mínima, resultando em tabelas dominadas por zeros.
Questão 07
A fórmula de suavização de Laplace para bigramas é:
Resposta: (b). Add-1: soma 1 ao numerador e |V| ao denominador para manter a normalização, pois cada bigrama possível ganhou +1. |V| = tamanho do vocabulário.
Questão 08
Em que tarefa os modelos de n-gramas detectam erros como "chutou a mola" (em vez de "bola")?
Resposta: (c). Erros contextuais são ortograficamente corretos mas semanticamente errados. O modelo de bigramas atribui P("mola" | "chutou a") muito menor que P("bola" | "chutou a"), identificando o erro.
Questão 09
Qual afirmação sobre avaliação de modelos de n-gramas é CORRETA?
Resposta: (c). Avaliar no corpus de treinamento causaria sobreajuste. A divisão típica é 80% treinamento / 20% teste (ou 70/15/15 incluindo validação).
Questão 10
"Colorless green ideas sleep furiously" (Chomsky, 1955) é relevante para modelos de n-gramas porque:
Resposta: (b). Chomsky criou a frase para mostrar que gramaticalidade ≠ sensatez. Modelos de n-gramas podem atribuir P > 0 à frase se os bigramas individuais existirem no treino, sem capturar a incoerência semântica global.
Capítulo 4 – Classificadores bayesianos ingênuos
Questão 01
A "ingenuidade" do classificador bayesiano ingênuo refere-se à suposição de que:
Resposta: (b). O classificador "ingênuo" assume independência condicional: P(f₁, f₂,…,fₙ | c) = ∏ P(fᵢ | c). Raramente verdade na língua, mas simplifica o cálculo e produz bons resultados na prática.
Questão 02
Na avaliação de um classificador, a Precisão (Precision) é definida como:
Resposta: (b). Precisão = VP / (VP + FP): dos itens classificados como positivos, qual fração é de fato positiva. Cobertura = VP / (VP + FN). Acurácia = (VP + VN) / total.
Questão 03
Por que somar logaritmos de probabilidades em vez de multiplicá-las no classificador bayesiano?
Resposta: (b). Multiplicar centenas de probabilidades entre 0 e 1 resulta em números menores que o mínimo representável por float (underflow). Como log(a×b) = log a + log b, somar logs é equivalente e numericamente estável.
Questão 04
Qual a diferença entre Cobertura (Recall) e Acurácia?
Resposta: (b). Cobertura = VP / (VP + FN): dos positivos reais, quantos foram detectados? Acurácia = (VP + VN) / total: taxa geral de acertos. Acurácia pode ser enganosa em datasets desbalanceados.
Questão 05
A Medida-F combina precisão e cobertura porque:
Resposta: (b). F = 2 × (P × R) / (P + R). Um classificador pode ter alta precisão sendo muito conservador (muitos FN) ou alta cobertura sendo permissivo (muitos FP). A Medida-F penaliza ambos os extremos.
Questão 06
No contexto de classificação de spam com Naïve Bayes, o que são "atributos" (features)?
Resposta: (b). Features são propriedades mensuráveis dos dados. Em PLN, atributos típicos incluem presença/ausência de palavras, n-gramas, POS tags, tamanho do texto, etc.
Questão 07
A Regra de Bayes P(A|B) = P(B|A) × P(A) / P(B) permite ao classificador:
Resposta: (a). Queremos P(classe | mensagem), mas é mais fácil estimar P(mensagem | classe). A Regra de Bayes inverte essa relação: P(c | d) ∝ P(d | c) × P(c), eliminando P(d) por ser constante para todas as classes.
Questão 08
A suavização com parâmetro k no classificador bayesiano ingênuo serve para:
Resposta: (b). Sem suavização, P(palavra | spam) = 0 se a palavra nunca ocorreu em spam no treinamento, zerando toda a probabilidade do documento. A suavização add-k redistribui massa probabilística para cobrir palavras ausentes.
Questão 09
Word Sense Disambiguation (WSD) é uma tarefa de PLN que consiste em:
Resposta: (b). WSD resolve ambiguidade lexical: "manga" pode ser fruta ou parte da roupa; "banco" pode ser instituição financeira ou assento. Classificadores bayesianos usam o contexto para selecionar o sentido correto.
Questão 10
Com 6.000 spams e 4.000 não-spams, a probabilidade a priori P(spam) estimada por MLE é:
Resposta: (c). P(spam) = N_spam / N_total = 6.000 / 10.000 = 0,60. A probabilidade a priori reflete a distribuição das classes no corpus de treinamento.

Fórmulas essenciais para memorizar

  • TF: freq. do termo no doc / total de termos no doc
  • IDF: log( N / df_t ) — N = total de docs, df_t = docs com o termo
  • TF-IDF: TF × IDF
  • MLE bigrama: P(w₂|w₁) = C(w₁,w₂) / C(w₁)
  • Laplace bigrama: (C(w₁,w₂) + 1) / (C(w₁) + |V|)
  • Perplexidade: PP = (1/P(seq))^(1/n) — quanto menor, melhor
  • Precisão: VP / (VP + FP)
  • Cobertura: VP / (VP + FN)
  • Medida-F: 2 × (P × R) / (P + R)
  • Acurácia: (VP + VN) / total

Dicas finais

  • Avaliação de modelos SEMPRE usa corpus de TESTE separado do treinamento
  • Perplexidade baixa = modelo melhor; perplexidade alta = modelo pior
  • Naïve Bayes assume independência condicional entre features (daí o "ingênuo")
  • TF-IDF: palavras muito frequentes em todos os docs têm IDF ≈ 0 → peso baixo
  • Hápax legômena afetam modelos de n-gramas → problema de dados esparsos
  • Suavização (Laplace / add-k) evita probabilidade zero em ambos os modelos
